# Mobile App Manual Testing

This project contains manual test scenarios designed to validate the functionality and user experience of a mobile application.

The focus is on testing real user flows, UI behavior, and identifying functional issues.

---

## Objective

To ensure the mobile application works correctly from the user's perspective, covering both functional behavior and usability.

---

## What Was Tested

- Order creation flow  
- Selection of pickup location  
- Product selection and cart behavior  
- Order confirmation process  
- Navigation between screens  
- UI consistency and layout  

---

## Test Approach

- Manual test execution  
- Positive and negative scenarios  
- UI/UX validation  
- Functional testing of user flows  
- Identification of inconsistencies and unexpected application behavior 

---

## Key Findings

During testing, several issues were identified:

- UI elements overlapping in some screens  
- Incorrect total price calculation in certain scenarios  
- Inconsistent language or labels in the interface  
- Minor layout and usability issues affecting user experience  

---

## Tools Used

- Mobile application testing  
- Manual testing techniques  
- Test case documentation

This project was developed as part of a QA engineering bootcamp.
